Blast from the Past! ‚ö°

Long before computers used binary, another code system was used to send messages across vast distances: Morse Code!

Just like how computers use 1s and 0s to represent letters, Morse Code used dots (.) and dashes (-) to represent each letter of the alphabet. This was one of humanity's first attempts at converting text into a different format that could be easily transmitted!

For example: SOS = ... --- ...

Just like Morse Code revolutionized long-distance communication, binary code revolutionized how computers process information.

There are actually lots of similarities between Morse Code and binary. Let's see how computers do it!

How Do Computers Store Text? üìù

Computers can only understand numbers - specifically, binary (1s and 0s). To store text, we need a way to convert letters into numbers. That's where ASCII comes in!

We use a byte (a set of 8 bits) to represent different characters, and even things like spaces and full stops.

For example, in ASCII:

  • The letter 'A' is stored as 01000001
  • The letter 'B' is stored as 01000010
  • The letter 'C' is stored as 01000011

Beyond ASCII: Unicode üåç

ASCII only handles English letters and basic symbols. But what about other languages and symbols? That's where Unicode comes in!

ASCII was originally developed for a limited amount of characters and symbols in English. Remember - we can only represent up to 255 values with a single byte!

Unicode is the answer to this. Because we have more storage and computing power, we can agree to use from 2 to 4 bytes to represent more characters and symbols.

Remember that our available numbers grow rapidly as we add more bits. With four bytes (or 32 bits) we can represent over 4 billion characters or symbols!
