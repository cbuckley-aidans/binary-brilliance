How Computers See Text

Computers only understand numbers - specifically, binary numbers made up of 1s and 0s. They can't directly understand letters, symbols, or emoji the way humans do.

1

What is Binary?

Binary is a number system that only uses two digits: 0 and 1.

Each 0 or 1 is called a bit (binary digit).

01000001 = The letter "A"

Binary is the language of computers because electronic components can easily represent two states: ON (1) and OFF (0).

2

Why We Need Text Encoding

For computers to work with text, we need a system that converts between:

Hello!
01001000 01100101 01101100...

This conversion system is called text encoding. It's like having a special code book that tells the computer which binary numbers represent which characters.

3

Bytes: Grouping Bits

A single bit (0 or 1) doesn't give us many options - just two! To represent more information, computers group bits together into bytes.

0 1 0 0 0 0 0 1

1 byte = 8 bits

With 8 bits in a byte, we can represent 256 different values (from 0 to 255). That's enough for all the letters, numbers, and basic symbols we need!

Key Points

COMPUTERS SPEAK BINARY

Computers only understand binary (1s and 0s)

TEXT NEEDS ENCODING

We need a system to convert between text and binary

BYTES STORE CHARACTERS

Each character is typically stored as one byte (8 bits)

ASCII: The Text Encoding System

In the early days of computing, a system called ASCII (American Standard Code for Information Interchange) was created to standardize how text should be represented in binary.

1

How ASCII Works

ASCII assigns a unique number to each character:

  • 'A' is assigned the number 65
  • 'B' is assigned the number 66
  • 'a' is assigned the number 97
  • '1' is assigned the number 49
  • Space is assigned the number 32

These numbers are then converted to binary for the computer to process!

2

ASCII in Binary

Since ASCII uses numbers from 0 to 127, we only need 7 bits to represent all ASCII characters. But computers typically use 8 bits (1 byte) for convenience.

Character ASCII Value Binary
A 65 01000001
B 66 01000010
a 97 01100001
Space 32 00100000
3

TEACHER CHUNK

From ASCII to Unicode: Supporting Global Text

ASCII was great for English, but has a major limitation - it only supports 128 characters! This isn't enough for other languages and symbols.

1

The Limitations of ASCII

ASCII can only represent:

English Letters

A-Z, a-z

Numbers

0-9

Basic Symbols

! @ # $ % ^ & * ( ) etc.

But ASCII cannot represent:

Other Languages

Chinese, Arabic, Russian

Emojis

😊 🎮 🚀

Special Symbols

∑ ∞ ♠ ♥ ♦ ♣

2

Unicode to the Rescue

Unicode was created to solve ASCII's limitations. Instead of just 7-8 bits, Unicode uses multiple bytes to represent characters from all languages and systems.

Unicode can support over 1.1 million characters! That's enough for all human languages - even ancient ones.

你

Chinese Character "Nǐ" (You)

Unicode: U+4F60

Binary: 11100100 10111101 10100000

Ж

Russian Letter "Zh"

Unicode: U+0416

Binary: 11010000 10010110

😊

Smiling Face Emoji

Unicode: U+1F60A

Binary: 11110000 10011111 10011000 10001010

Remember: What's a Byte?

A byte is a group of 8 bits. While ASCII uses 1 byte (8 bits) per character, Unicode characters can use 1 to 4 bytes depending on the character. This is why Unicode can represent so many more characters than ASCII!

3

UTF-8: The Most Common Unicode Format

There are different ways to encode Unicode characters in binary. The most common is UTF-8, which is used by most websites and systems today.

UTF-8's Smart Design:

  • Uses 1 byte for basic Latin/ASCII characters (compatible with ASCII)
  • Uses 2-4 bytes for other characters
  • Can represent any Unicode character

Making Digital Data Smaller

Data compression is the process of reducing file sizes by finding patterns and removing redundancy. It's essential for modern computing, making everything faster and more efficient.

1

Why Compress Data?

Digital files can be enormous! Compression reduces file sizes by finding patterns and removing redundancy, making storage and sharing much more efficient.

  • Save storage space on devices
  • Faster file downloads and uploads
  • Reduce internet bandwidth usage
  • Lower costs for cloud storage

Example: A 10MB photo can be compressed to 1MB while looking almost identical!

2

Types of Compression

Lossless

Perfect reconstruction: Original data can be completely recovered

Examples: ZIP files, PNG images, FLAC audio

Lossy

Some data lost: Better compression but quality reduced

Examples: JPEG images, MP3 audio, MP4 video

Trade-off: Lossy compression achieves smaller file sizes but loses some original data forever.

Compression vs Encryption

Compression and encryption serve different purposes, but both work with binary data. Let's understand the key differences:

Compression

Purpose: Make files smaller

Goal: Save storage space

Example: "the cat" → "@cat" (@ = "the")

Readable: Yes, once decompressed

Encryption

Purpose: Hide information

Goal: Protect data security

Example: "hello" → "khoor" (shift +3)

Readable: No, without the key

2

Caesar Cipher Practice

The Caesar cipher is one of the oldest encryption techniques. It shifts each letter by a fixed number of positions in the alphabet.

How Caesar Cipher Works:

A B C D E
↓ +3 ↓
D E F G H

Example: "HELLO" with shift +3 becomes "KHOOR"
